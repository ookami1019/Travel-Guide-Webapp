# エディタUI刷新：パワーポイント形式の実装計画

## 目的
ユーザーが印刷後の姿を常に意識しながら編集できるよう、エディタをA4用紙（スライド）単位のレイアウトに変更する。PCでの編集効率を最大化する。

## Proposed Changes

### [Layout] パワーポイント型 構成
- **トップリボン**: 「行程」「持ち物」などのブロック追加ボタンを上部に配置。アコーディオン形式で表示/非表示を切り替え可能。
- **左サイドバー**: 用紙（シート）の一覧サムネイルを表示。
- **メインキャンバス**: 選択されたA4用紙を中央に大きく表示。実際の印刷サイズ比率を維持。

### [Editor logic] ダイレクト編集への移行
- 現在の `ExportPreview` で行っていた面付け（Imposition）ロジックをエディタの基盤とする。
- ユーザーは「1ページ目（表紙）」「2ページ目（行程）」といったページ単位でコンテンツを保持。
- データ構造を `sheets: { pageLeft: Block[], pageRight: Block[] }[]` のような構成へリファクタリング。

### [Refactoring] 不要機能の削除
- 独立した「プレビューモード」を廃止し、編集画面そのものを最終成果物に近い状態にする。

## Verification Plan
1. トップリボンの開閉が正常に動作し、ブロック追加が可能か。
2. サイドバーでのシート切り替えと、中央キャンバスでの直接編集が可能か。
3. PDF出力時に、エディタの見た目通りの配置で出力されるか。
